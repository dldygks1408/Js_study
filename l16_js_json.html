JSON JavaScript Object Notation

  • 객체 표현법
  • JSON은 문자열로 표현 될 수 있다.
  • 아주 가벼운 경량 데이터 교환 방식으로 문자열로만 이루어진다.(모든 프로그래밍 언어에 독립적이다.)
  • 데이터를 주고 받기위해 사용된다.

JSON 작성법

  • JSON : "{}"
  • const jsonData = "{}";
  • const jsonData = {key : value, key2 : value2}; 이때 key의 이름은 중복 될 수 없다.
  • const jsonData = "{\"key\" : \"value\", \"key2\" : \"value2\"}";
    문자열로 이루어진 데이터 형식이기 때문에 기본데이터 타입(문자열, 숫자, 논리, null)과 배열을 제외하고 모두 문자열로 작성
  • JSON 형식의 문자열은 자바스크립트에서 JSON.parse()로 Object 타입으로 변경한다.
  • Object로 변경된 json의 key는 프로퍼티로 .으로 접근 가능하다. : jsonDate.key
  • 프로그래밍에서 key가 유동적으로 바뀔때는 ["key"] 방식으로 접근 가능하다. : jsonData["key"]

자바스크립트가 JSON을 대하는 태도

  • 자바스크립트에서 JSON은 Object()의 프로퍼티 설명서
  • new Object로 JSON을 생성 할 수 있다.
  • 자바스크립트에서 JSON은 기본데이터타입처럼 리터럴한 방식을 제공한다.
  • {key : value} : 리터럴 방식 (자바스크립트 문법에서 사용되는 JSON)
  • "{'key' : 'value'}" : 문자열로 표현된 JSON (서버에서 주고 받는 데이터)
  • 자바스크립트는 함수지향 언어로 함수를 객체로 선언 할 수 있다.
  • function person(name,age){ this.name = name, this.age = age}
    => new Person("이요한", 31) = person : {name:"이요한", age:36}
  • 자바스크립트의 객체의 프로퍼티 설명서가 문자열로 데이터 전송되는 것을 JSON이라 한다.

자바스크립트에서 JSON 사용하는 법

  • 작성법 1. function ObjectName(value,value2){ this.key = value; this.key2 = value2; }
  • 작성법 1.5 function ObjectName(key,key2){ this.key; this.key2; }
  • 작성법 2. (리터럴 방식) {key:value, key2=value2}
  • 작성법 2. (this 연산자를 바로 사용 불가): window 가리킴 {key:value, key2:value2, key3:this.key+this.key1} (X)
  • 리터럴방식의 json 선언시 this는 블럭 내의 객체를 가르키지 않고 window를 가르킨다.(화살표 함수랑 같다)
  • 작성법 this 사용법 (함수에서 this 사용시 블럭객체를 가르킨다.) {key:value, key2:value2, key3: function(){return this.key+this.key2}} (O)
  • 작성법 3. ( X 추천 X ) const ObjectName = new Object(); ObjectName.key = value; ObjectName.key2 = value2;
