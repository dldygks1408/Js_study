javascript class

  • let a = "안녕"; let a = new String('안녕'); 기본데이터 타입은 모두 리터럴한 방식
  • const arr = [1,2,3]; (리터럴한 선언 방식) / new Array(1,2,3)
  • object = {name:"yo", age : 30}
  • class Person{constructor()} => new Person()
  • class 를 선언하는 이유 (캡슐화) : 사용할 객체를 사용하지 않는것과 분리(은닉), 사용하는 객체를 문제가 생기지 않도록 처리, (속성)멤버변수를 선언하고 (행위)메서드를 선언하는 행위
  • 언어적 측면에서 접근 지정자에 의해 정의된 변수나 멤버 변수에 의해 오류가 발생 할 것을 사전에 방지하는 행위(name=null 기계적 오류 X)
  • "이_요한".split("_"), null.split("_") => 오류 위처럼 방지
  • class를 선언하는 이유 (재사용) : 상속으로 class를 재사용 가능
  • class를 선언하는 이유 (유지보수) : 선언된 class를 수정하면 참조하는 모든 class가 변경된다.(유지보수 용이)
  • class의 단점 : 설계시 개발 속도를 느리게 한다.
    객체가 많고 설계가 잘못되면 용량이 커진다.
    자바스크립트는 class를 function의 대용품으로 사용중(함수가 클래스인 척 하고 있다.)
