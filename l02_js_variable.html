자바스크립트의 변수(variable)

모든 프로그래밍 언어는 데이터를 참조하기 위해 변수를 사용한다.
이때 참조하는 데이터가 변할 수 있어서 변수라 한다. (let a = "안녕" -> a = "하이")

자바스크립트의 변수와 상수

  • var : 자바스크립트 태초의 변수로 함수지향언어의 특징을 가진 블럭이 모호한 상태다.(사용하지 않음!)
  • let : ES6에 탄생한 변수로 클래스 추가와 함께 블럭의 구분을 명확하게 한다.(var 대신 사용)
  • const : 변수가 아닌 상수로 선언과 동시에 데이터를 참조해야 하고 참조된 데이터를 변경 할 수 없다.
    참조하는 데이터의 상태를 변결 할 수 있다.

변수의 식별자(이름)

프로그래밍에서 식별자 즉 이름 짓기는 아주 중요하다. 유지보수에 용의하기 위해 약속된 방식이지만
이름이 오류를 만들 수 있어 오류가 있는 코드를 배포하는 자바스크립트의 특성상 이름 짓기가 더 중요하다.
예) let var = 10; let function = "test" 오류 : 예약어 사용 불가

  • 변수 이름은 문자, 숫자, 밑줄(-제외), $ 가 포함 될 수 있다.
  • 변수는 숫자로 시작하는 것을 권장하지 않는다. 예) let 1-num (x) let num_1 (o)
  • 변수의 이름은 구체적이여야하고 문자와 문자의 결합을 카멜표기법(낙타)or _로 구분한다. 예) javaScript, memNum, userId (카멜표기법) java_script, mem_num, user_id (_표기법)
  • 상수(const)의 이름은 모두 대문자이고 _표기한다.(COLOR_RED)
  • 이름은 대소문자를 구분한다. (num!=Num!=NUM 모두 다른 변수)
  • 예약어는 변수 이름으로 사용 할 수 없다, (function, var, let Number...)

식별자에 그 밖의 규칙(권장)

  • 읽고 이해 할 수 있는 이름을 사용(userName, colorName, colorHexCode)
  • a, b, c , i 같은 짧은 이름은 피하고, 구조적으로 변수명이 명확할 때 사용가능(for(i->index), mem(num))
  • 소속된 팀의 규칙을 따른다. (userId, memberNum) x , (userId, userNum) o 통일성
