JavaScript JS 소개

자바스크립트는 웹 브라우저 상에서 UI를 동적으로 보여주기 위하여 개발되었으나 배우고 쉽고 개발 속도가
빨라서 서버(Node.js)나 모바일앱(react-native, NativeScript)등 프로그래밍의 여러 분야에서 사용되고 있다.
또한 매해 새 버전을 출시해 오류를 수정하거나 객체지향적 특성을 발전 시키고 있다.

브라우저 언어인 자바스크립트의 특징

  • 자바스크립트는 객체기반의 스크립트 언어로 브라우저에서 사용된다.
    ( 스크립트 언어 : 응용프로그램을 동작시키는 언어의 통칭 )
  • 자바스크립트는 인터프리터언어로 컴파일이 되지않고 바로 해석된다.
    ( 컴파일 : 작성한 코드를 기계어로 해적하는 행위, 오류가 있는 코드는 컴파일 되지않는다. )
    때문에 동적으로 개발 할 수 있지만 실행 속도가 느리고 오류가 있는 페이지를 배포 할 수 있다.
  • 자바스크립트는 함수형 프로그래밍 언어였으나 ES6 이후로 class와 변수를 추가해
    객체지향 언어의 특징을 가진다.
  • 자바스크립트는 주로 다루는 객체가 window.document로 DOMtree로 구성된 document의
    특정 Element를 찾는 것에서 부터 시작된다. 예) document.getElementById()
  • 자바스크립트는 html 문서가 모두 브라우저에 출력되고 마지막에 실행된다.

자바스크립트가 브라우저에서 할 수 있는 일

  • html 문서의 내용과 속성을 추가, 수정, 삭제가 가능하다.
  • html 문서의 스타일을 변경 할 수 있다.
  • html 요소에서 발생하는 외부기기의 입력을 이벤트로 받아 UI를 동적으로 바꾼다.
  • 브라우저의 (BOM) 정보를 알 수 있거나 입력 및 수정을 할 수 있다.

html에서 자바스크립트 작성법

  • Script 코드는 script 태그 내부에 작성해야한다.
  • script type="text/JavaScript" 올드 자바스크립트로 사용되지 않는다. type 생략
  • script src=" .js" 확장자가 js인 파일을 외부에서 참조 할 수 있다.
  • 코드를 실행 및 선언 후 ";" 세미콜론을 꼭 작성해야한다.
  • // script 주석
  • /* */ script 여러줄 주석
  • "use script" : 엄격모드 (ES6 문법 사용시 자동으로 설정, class let...)
    js 파일 상단에 꼭 작성해야 한다. 과거 문법이 오류를 많이 만들기 떄문이다.
    예 ) var num=10; (O) num=10; (O,X)
